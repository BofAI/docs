# 身份认证

## 谁可以在 ERC-8004 上注册？
该协议是为 Agents 设计的，更广泛地说，是为任何 API 或服务设计的（我们相信“Agents”与“数字服务”/API 之间的界限在未来几年将逐渐模糊）。该协议是无许可的 (permissionless)，因此注册一个 Agent 只需要一个钱包来签署交易并支付**能量和带宽** (energy and bandwidth)。我们预计各种平台会**赞助能量和带宽**，以使体验更加顺畅。

## 如何注册？
一旦你选择了在哪个区块链上注册（例如**以太坊主网**），你可以使用可用的 SDK 之一，或使用你自己的代码进行注册——这相当于铸造一个 NFT（我们使用 ERC-721 标准）。

从技术上讲，注册是一个链接到该 Agent（拥有 `agentId`）的 NFT，它包含一个指向注册文件 `agent-registration.json` 的链接 (`agentUri`)。任何人都可以访问该文件，其中展示了关于你的 Agent 的信息：名称、图像、功能及所售服务的描述、接收付款的区块链地址 (`agentWallet`) 以及联系方式 (`endpoints`)。

## 支持哪些端点 (Endpoints)？
一个 Agent 可以拥有一个或多个端点，具体由你决定。建议的端点包括：

- Web（供人类使用的简单 URL）
- MCP
- A2A
- 任何 DID
- ENS 域名
- 电子邮件地址

但此列表可以扩展以支持新的端点类型。

## 注册文件存储在哪里？
存放在你偏好的任何位置。`agentUri` 可以指向托管在你服务器上并通过 HTTPS 提供的普通 JSON 文件，也可以指向去中心化基础设施 (IPFS) 上的文件，或者可以直接包含作为 Base64 编码的二进制字符串并存储在链上的文件。

## 如何验证端点所有权？
验证是可选的，因此在 ERC-8004 上，任何人都可以注册属于他人的 Agents 和服务（即非其所有的）。如果你想证明你是该 Agent 的所有者——即验证端点所有权：

- **如果端点域名与托管注册文件的域名相同（HTTPS 模式）** → 域名将自动通过验证。
- **对于所有其他域名** → 你可以通过将注册文件的副本（或按规定仅包含 "registrations" 部分）保存至以下路径来验证域名：`domain.com/.well_knowns/agent-registration.json`

## 所有权和可转移性是如何处理的？
Agents——作为各方面都符合 ERC-721 标准的代币 (NFT)——拥有一个所有者和一个或多个操作员。所有者可以随时将 Agent 转移到另一个地址（使用标准转移功能）。

在转移时，`agentWallet` 字段会被重置，新所有者可以验证一个新的地址来接收付款（可选择新所有者自己的地址）。
