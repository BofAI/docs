# 简介

## 什么是 ERC-8004？
ERC-8004 是一个开放协议。它允许你注册自己的 Agents 和 API，使其具备可见性和可移植性；允许你发现他人的 Agents 和 API；还能让你根据过往客户的反馈以及第三方的验证结果，来决定选用哪一个。

简而言之：**它是一个 Agents 发现与建立信任的协议。**

## 我为什么要使用它？
- **Agents 开发者 (Agent builder)：** 增加你的 Agents/服务的可发现性和可移植性，这意味着获得更多的曝光、用户和客户。
- **用户 (User)：** 用于查找可用的 Agents 和服务，并判断哪些值得信任。
- **平台构建者 (Platform builder)：** 利用成百上千个 Agents 的注册和信任数据来构建浏览器、平台和市场，而无需从零开始。
- **金融/声誉研究 (Finance/reputation)：** 基于 ERC-8004 的公共数据库构建排名和评分算法。
- **研究人员 (Researcher)：** 在开放的 8004 数据集上进行研究（未来甚至可以进行模型后训练）。

## 它的工作原理是什么？
ERC-8004 利用区块链作为任何人都可以读写的共享公共注册表，包含：
- Agents 身份（身份注册表 / Identity Registry）
- 关于其他 Agents 和服务的反馈（信誉注册表 / Reputation Registry）
- 验证其他 Agents 和服务行为的加密证明（验证注册表 / Validation Registry）。
  > **注意：** 验证注册表仍在进行技术尽职调查，暂不可用。

## 它兼容哪些区块链？
ERC-8004 旨在成为区块链上 Agents 与服务发现及信任的通用标准。它目前可在主要的 EVM（以太坊虚拟机）链上使用，并将很快在一些非 EVM 链上推出。

虽然合约是开源的，任何人都可以在自己的链上部署 ERC-8004，但为了协议架构的统一性，每条链上最好只有一个部署（单例模式）。因此，如果你想在你的链上部署该协议，我们邀请你与 ERC-8004 团队协调。

## ERC-8004 的使命和影响是什么？
我们买卖的许多产品和服务将由 AI 驱动的软件（如聊天机器人）或自主 Agents 作为中介。如果存在中介寡头垄断（就像移动应用只能通过两个应用商店购买那样），可能意味着对全球 GDP 中日益重要的份额进行审查和费用抽取。

开放的发现与信任协议可以避免这种情况，同时也让初创公司和新入局者更容易提供他们的服务和 Agents。

## 为什么我们需要区块链？
使用中心化技术意味着由一家公司（受特定司法管辖区管辖）控制身份和信誉注册表——从而决定谁在注册表中、谁不在，谁获得曝光，服务的评分是多少，以及是否（以及与哪些组织）分享原始信誉数据。

区块链上的智能合约是避免这种情况的技术工具：Agents 构建者和用户将其数据交给一个**可信的中立基础设施**，其行为由智能合约中的代码描述。这就是使 ERC-8004（及其数据库）成为 Public Good 的原因。
