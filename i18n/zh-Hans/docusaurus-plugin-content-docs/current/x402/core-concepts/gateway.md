# 网关

将任意 API 转化为面向 AI Agent 的按次计费服务。

网关接在你的 API 前面，负责处理收款：AI Agent 自主发现、调用并完成付费，每一笔调用都在链上结算，款项直接进入你指定的钱包。现有 API 保持原样，无需改动任何代码。

:::tip
想直接上手？跳到 [上架你的 API](../../agent-apis/list-your-service.md)；想先看看有哪些服务，去 [浏览 API 目录](../../agent-apis/index.md)。
:::

## 它是怎么工作的

可以把网关理解为站在你 API 前面的**收银台 + 中转站**（技术上是一层反向代理）：

- **Agent 只访问网关地址**（如 `https://gateway.../providers/<fqn>/v1/...`），不会直接触达你的上游 API。
- **网关负责转发**。验款通过（或端点免费）后，网关把请求代理到上游，再把上游结果原样返回给 Agent。
- **凭证隔离**。上游 API Key 等敏感配置只保存在网关侧的 `provider.yml` 或环境变量中，调用方全程接触不到。
- **收不收费由你按端点决定**。在配置中为端点定价即走付费流程；不定价（价格为 0）的端点直接转发，免费接口依然免费。

```text
Agent ──► 网关 ──► 你的上游 API
          │ 报价 / 验款 / 结算（仅付费端点）
          └ 帮你转发请求、代管上游密钥
```

## 一次调用的完整流程

### 付费端点：HTTP 402 流程

每一笔付费调用都通过 x402 在链上清算——响应中返回的报价始终是唯一真实价格。

1. **Agent 发起调用** —— 请求目标接口。
2. **网关报价** —— 返回价格（HTTP `402`）。
3. **钱包付款** —— Agent 在链上完成支付。
4. **验款并结算** —— 资金结算至你的收款钱包。
5. **返回结果** —— 网关把请求转发到上游 API，将结果返回给 Agent。

### 免费端点：直接转发

未定价的端点没有报价环节，也不发生任何链上交易：

1. **Agent 发起调用** —— 请求目标接口。
2. **网关直接转发** —— 代理到上游 API，把结果返回给 Agent。

也就是说，同一个服务可以付费端点与免费端点并存——比如 `/v1/current` 按次收费，而 `/v1/ping` 保持免费。免费端点在目录里也会标示出来，Agent 一眼就能看到哪些能力不花钱。

:::note
无论付费与否，上游凭证始终保留在网关侧，调用方 Agent 全程无法接触任何密钥。
:::

## 它给两边带来什么

提供方将每一次调用转化为现金流，Agent 将每一次调用转化为能力，网关连接供需两端。

### 面向 API 提供方：上架 API，让每一次调用都产生收入

- **资金直达钱包** —— 款项实时结算至你的收款钱包，无平台托管、无结算周期。
- **原生多链** —— 可在 TRON 或 BNB Chain 上结算，由你自主选择。
- **零代码改动** —— 现有 API 保持原样，无需改动任何代码。
- **定价自主** —— 可按接口、按档位灵活定价，免费接口依然免费。

→ [上架你的 API](../../agent-apis/list-your-service.md)

### 面向 AI Agent：一个目录，汇聚海量 API，按需付费

- **MCP 原生发现** —— 按名称精准定位并调用所需 API。
- **按次计费** —— 无需订阅，无最低消费。
- **实时价格透明** —— 每次调用的价格均由网关实时签发，所报即所付。
- **无账号、无密钥** —— 钱包即身份，无需注册或管理任何 API Key。

→ [浏览 API 目录](../../agent-apis/index.md)

## 怎么把你的 API 接进来

三步即可上线，现有 API 无需任何改动：

1. **选网关** —— 用 BANK OF AI 官方托管的网关，或自己部署一个（两种方式的区别见下）。
2. **定好价目表** —— 确定接口、各端点定价与收款钱包；无需集成 SDK，无需重写任何代码。
3. **上架目录** —— 向目录仓库提交两份公开文件（可先填表申请，由我们联系协助）。服务出现在 [API 目录](../../agent-apis/index.md) 后，任意 Agent 都能发现、调用并自动付费。

完整步骤见 [上架你的服务](../../agent-apis/list-your-service.md)。

### 官方网关，还是自己运行？

- **官方网关（托管）**：由 BANK OF AI 运行网关，代理你的 API 并处理收款，你无需部署任何东西；上架文件里的调用地址填官方域名（形如 `gateway.bankofai.io/providers/<fqn>`）。
- **自己运行网关**：在自己的机器上部署网关，上游凭证与收款配置只留在本地；上架文件里的调用地址填你自己的域名。适合希望完全自主掌控凭证与基础设施的服务方。

无论哪种方式，调用方 Agent 的体验完全一致：发现、报价、付款、调用走的都是同一套 x402 流程。

:::note 自建网关了，为什么还要上架目录？
网关和目录解决的是两件事：**网关负责收钱，目录负责让你被发现**。目录只是一份公开的服务清单，不经手任何调用与资金——Agent 调用的永远是目录里登记的网关地址。自建网关时，登记的就是你自己的域名，流量与款项直接到你；上架的意义在于：所有接入 BANK OF AI 目录的 Agent、CLI 用户与目录站访客都能发现并调用你的服务，而不只是知道你域名的人。
:::
