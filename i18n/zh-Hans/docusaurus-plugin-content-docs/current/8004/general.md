# 简介

## 什么是 8004 协议？
在 Web3 环境下，智能体（Agent）的身份与信誉是其最核心的数字资产，而 8004 协议正是这些资产的守护者与分发者。在 TRON 网络上，8004 协议基于原生 TRC-8004 标准深度构建；在 BSC 网络上，则复用了 ERC-8004 的标准实现。这种多链同构的设计，确保了智能体身份与信誉在不同主流网络间的一致性。它允许你注册自己的 Agents 和 API，使其具备可见性和可移植性；允许你发现他人的 Agents 和 API；还能让你根据过往客户的反馈以及第三方的验证结果，来决定选用哪一个。

简而言之：**它是一个 Agents 发现与建立信任的协议。**

## 工作原理
8004 协议利用区块链作为任何人都可以读写的共享公共注册表，包含：
- Agents 身份（身份注册表 / Identity Registry）
- 关于其他 Agents 和服务的反馈（信誉注册表 / Reputation Registry）
- 验证其他 Agents 和服务行为的加密证明（验证注册表 / Validation Registry）。


## 核心概念
- **身份认证**
  
  8004 协议建立了一套无许可的身份认证体系，允许任何 Agent 或服务通过铸造兼容 TRC-721/ERC-721 标准的 NFT 完成注册，其核心是将链上身份 (agentId) 映射至存储于任意位置（HTTPS、IPFS 或链上）的元数据文件 (agent-registration.json)。该文件定义了 Agent 的基本信息、支付钱包及多样化的通信端点（如 Web、MCP、DID 等），并支持通过域名自动匹配或 .well-known 路径验证端点所有权。作为 NFT 资产，Agent 身份具备完全的可转移性，且在所有权转移时会自动重置支付地址，从而确保新所有者对服务收入的安全控制。
- **信誉系统**
  
  8004 协议定义了一个中立的公共信誉存储标准，通过结构化的签名数据支持对 Agent 进行多维度反馈，并将具体的信誉算法、排名聚合及反垃圾过滤逻辑完全交由生态应用层自主构建。
- **共识验证**  
  
  验证注册表通过智能合约集成TEE证明、加密经济质押及zkML等多种技术手段，实现了对Agent行为的可委托加密验证。
