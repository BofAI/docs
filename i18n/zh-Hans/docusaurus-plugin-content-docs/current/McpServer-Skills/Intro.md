# 简介

BANK OF AI 通过 MCP Server 与 Skills 的双层架构，为构建基于区块链网络的自主智能体（AI Agents）提供了从基础设施 to 业务逻辑的完整解决方案。


## MCP Server 

MCP Server 旨在为大型语言模型 (LLM) 与外部工具和数据源之间的交互提供一个标准化的框架。其核心在于一个清晰定义的客户端-服务器架构和分层通信机制，确保了 AI 应用能够安全、高效地访问和利用外部信息。

## 1. 架构参与者

MCP 遵循**客户端-服务器**架构，主要包含以下三个核心参与者：

*   **MCP 主机**：通常是 AI 应用程序本身（例如 Claude Code 或 Claude Desktop）。它负责协调和管理一个或多个 MCP 客户端。
*   **MCP 客户端**：由 MCP 主机为每个 MCP 服务器实例化。每个客户端维护与对应 MCP 服务器的专用连接，并从服务器获取上下文信息供主机使用。
*   **MCP 服务器**：一个程序，负责向 MCP 客户端提供上下文数据。服务器可以是本地运行的（例如文件系统服务器），也可以是远程运行的（例如 Sentry MCP 服务器）。

这种架构允许 AI 应用通过多个客户端同时连接到不同的 MCP 服务器，从而扩展其能力。

## 2. 分层结构

MCP 协议由两个主要层组成 [1]：

*   **数据层 (Data Layer)**：这是 MCP 的核心，定义了基于 **JSON-RPC 2.0** 的客户端-服务器通信协议。它涵盖了生命周期管理、核心原语（工具、资源、提示）以及通知机制。数据层关注消息的结构和语义。
*   **传输层 (Transport Layer)**：负责管理客户端和服务器之间的通信通道和认证。它处理连接建立、消息帧处理和安全通信。传输层抽象了通信细节，允许数据层在不同的传输机制上使用相同的 JSON-RPC 2.0 消息格式。

### 传输机制

MCP 支持两种主要的传输机制 ：

*   **标准输入/输出 (Stdio) 传输**：适用于同一机器上本地进程间的直接通信。它利用标准输入/输出流，提供最佳性能且没有网络开销，通常用于本地 MCP 服务器。
*   **可流式 HTTP (Streamable HTTP) 传输**：通过 HTTP POST 进行客户端到服务器的消息传输，并可选择使用 Server-Sent Events (SSE) 实现流式传输。这种机制支持远程服务器通信，并支持标准的 HTTP 认证方法（如 Bearer Token、API 密钥等），推荐使用 OAuth 获取认证令牌。

### 3.1 初始化交换 (Initialization Exchange)

这是 MCP 通信的起点，通过能力协商握手建立连接。客户端向服务器发送 `initialize` 请求，以协商支持的协议版本和功能。服务器响应客户端的能力，从而建立一个兼容的通信会话。这个过程确保了客户端和服务器能够理解彼此的能力并进行有效通信。

### 3.2 工具发现 (Tool Discovery)

一旦连接建立，AI 应用程序需要了解 MCP 服务器提供了哪些功能。客户端会使用 `*/list` 方法（例如 `tools/list`、`resources/list`、`prompts/list`）来发现服务器公开的所有工具、资源和提示。这种动态发现机制允许服务器根据需要提供不同的功能集。

### 3.3 工具执行 (Tool Execution)

当 AI 应用程序决定执行某个操作时，它会通过 MCP 客户端向服务器发送工具执行请求（例如 `tools/call`）。请求中包含要调用的工具名称和必要的参数。服务器接收请求，执行相应的操作（例如查询区块链数据），然后将结构化的结果返回给客户端。AI 应用程序随后解释这些结果并将其整合到其响应中。

### 3.4 通知 (Notifications)

MCP 支持实时通知机制，以实现客户端和服务器之间的动态更新。例如，当服务器提供的工具集发生变化（新增功能或修改现有工具）时，服务器可以发送工具更新通知给所有连接的客户端。通知以 JSON-RPC 2.0 通知消息的形式发送，不期望响应，确保了客户端能够及时了解服务器状态的变化。

## 4. 核心概念

MCP 定义了服务器可以公开的三种核心原语 [1]：

*   **工具 (Tools)**：可执行函数，AI 应用可以调用它们来执行文件操作、API 调用、数据库查询等具体动作。
*   **资源 (Resources)**：数据源，为 AI 应用提供上下文信息，例如文件内容、数据库记录、API 响应等。
*   **提示 (Prompts)**：可重用的模板，用于构建与语言模型的交互，例如系统提示或少样本示例。

此外，MCP 还定义了客户端可以公开的原语，例如**采样 (Sampling)**（允许服务器请求 LLM 完成）、**启发 (Elicitation)**（允许服务器请求用户输入）和**日志 (Logging)**（允许服务器向客户端发送日志消息），以构建更丰富的交互 [1]。

## 总结

开发者只需部署 MCP Server 并加载相应的 Skills，即可让 AI 智能体从“聊天机器人”进化为具备资产管理能力与链上交互能力的 Web3 经济实体。
