
# 简介

TRON MCP Server 旨在为 AI 代理提供与 TRON 区块链交互的能力。通过统一的接口，AI 代理可以利用工具和 AI 引导的提示来访问 TRON 网络服务，支持 TRX、TRC20 代币以及智能合约操作。该服务器利用 `tronweb` 库，全面支持 TRON 生态系统。

## MCP 服务器 URL
为了方便不同需求的用户，你可以选择直接连接[官方云服务](./UsingOfficialService.md)，也可以选择在自己的电脑上进行[本地化部署](./UsingLocalService.md)。其中官方服务只提供读取服务。

## 核心能力概览

*   **区块链数据**：读取区块、交易和链参数（如能量/带宽成本）。
*   **智能合约**：与任何 TRON 智能合约进行交互（支持读/写）。
*   **代币服务**：转移 TRX 和 TRC20 代币；检查余额及元数据。
*   **地址管理**：在 Hex 和 Base58 格式之间转换并验证地址。
*   **安全集成**：支持私钥和助记词 (BIP-39) 钱包，确保操作安全。
*   **多网络支持**：全面覆盖主网、Nile 和 Shasta 测试网。


## 详细功能特性

TRON MCP Server 提供的具体功能列表，供开发者和高级用户参考：

### 区块链数据访问
*   **多网络支持**：支持 Mainnet、Nile 和 Shasta 网络。
*   **链信息查询**：获取当前区块高度、链 ID 及 RPC 节点信息。
*   **区块数据**：通过区块号或哈希值访问详细区块信息。
*   **交易详情**：获取交易的完整细节，包括资源消耗（能量/带宽）。
*   **资源成本查询**：实时查询链上能量和带宽的价格参数。

### 代币服务
*   **原生 TRX**：支持余额查询和转账操作。
*   **TRC20 代币**：
    *   查询代币余额。
    *   执行代币转账。
    *   获取代币元数据（名称、符号、精度）。

### 地址服务
*   **格式转换**：在 Hex (`41...` 或 `0x...`) 与 Base58 (`T...`) 格式间无缝转换。
*   **地址验证**：验证特定地址在 TRON 网络上的有效性。

### 智能合约交互
*   **读取合约**：调用 `view` 和 `pure` 类型函数。
*   **写入合约**：执行改变链上状态的合约函数。
*   **ABI 自动获取**：自动从区块链获取已验证合约的 ABI 接口。

### 钱包与安全
*   **灵活配置**：支持通过 `TRON_PRIVATE_KEY` 或 `TRON_MNEMONIC` 进行配置。
*   **分层确定性 (HD) 钱包**：支持 BIP-44 派生路径 `m/44'/195'/0'/0/{index}`。
*   **消息签名**：支持对任意消息进行安全签名。

## 安全注意事项
*   **私钥管理**：私钥和助记词应始终通过环境变量安全管理，切勿硬编码或存储在不安全的文件中。
*   **测试网优先**：在将任何操作部署到主网之前，务必在尼罗河或沙斯塔测试网上进行充分测试。
*   **最小权限原则**：为 AI 代理配置的钱包应仅包含执行其任务所需的最低资金。
